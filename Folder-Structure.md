# Folder Structure

This document describes the **current** structure of `aws-kamel-app` and the module boundaries we enforce.

## Current repository layout

```text
aws-kamel-app/
  README.md
  Folder-Structure.md

  deploy/
    sam/
      kalmel-app-infra/
        template.yaml
        samconfig.toml
        Makefile
        README.md
        .aws-sam/                # generated build output (not source)

  services/
    lambdas/
      latest-state-writer/
        go.mod
        go.sum
        Makefile
        cmd/
          handler/
            main.go
        internal/
          processor/
            processor.go

      alarm-processor/
        go.mod
        go.sum
        Makefile
        cmd/
          handler/
            main.go
        internal/
          processor/
            processor.go
```

## Module model

Each Lambda service is an independent Go module.

- `services/lambdas/latest-state-writer/go.mod`
- `services/lambdas/alarm-processor/go.mod`

Implications:

- There is no root `go.mod` in the repository.
- Root `go.sum` was removed along with the root module and is not expected.
- Dependencies are managed per service.
- Build/test commands should run from each service directory (or explicitly target service paths).

## Deployment wiring model

SAM wiring lives under:

- `deploy/sam/kalmel-app-infra/template.yaml`

This template references service code using `CodeUri` paths:

- `../../../services/lambdas/latest-state-writer/`
- `../../../services/lambdas/alarm-processor/`

Build is delegated to each service Makefile via `BuildMethod: makefile`.

## Service boundaries

Rules we follow:

- `cmd/handler/main.go` is the service entrypoint.
- `internal/` contains service-private implementation details.
- Services must not import from other service folders.
- Shared code should be introduced under a future `libs/` directory if reuse appears.

## Build and test conventions

From `deploy/sam/kalmel-app-infra`:

```powershell
sam build
sam validate --lint --parameter-overrides Environment=dev
sam deploy --parameter-overrides Environment=dev
```

From each Lambda service folder:

```powershell
go test ./...
```

## Notes on generated artifacts

- `.aws-sam/` is generated by SAM CLI and should be treated as build output.
- Do not treat generated files as source-of-truth for architecture.

## Future growth

If new workloads are added:

- Add more Lambda modules under `services/lambdas/<name>/`.
- Add additional SAM apps under `deploy/sam/<app-name>/` when needed.
- Introduce `libs/` only when code is genuinely shared across services.
